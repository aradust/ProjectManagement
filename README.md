# Project Management

Веб-приложение для управления проектами, сотрудниками, задачами и связанными документами. Проект реализован как fullstack-решение: фронтенд на React и бэкенд на ASP.NET Core Web API с JWT-аутентификацией, ролевой моделью доступа и SQLite в качестве базы данных.

## Назначение проекта

Система помогает:

- создавать и сопровождать проекты;
- назначать менеджера и команду проекта;
- вести список сотрудников и управлять ролями;
- создавать задачи и отслеживать их статусы;
- прикреплять и просматривать документы по проектам;
- разделять доступ к функциональности по ролям пользователей.

## Основные возможности

### Аутентификация и роли

- регистрация и вход по email/password;
- JWT-авторизация;
- роли `Chief`, `Manager`, `Employee`;
- автоматическое создание главного пользователя (`Chief`) при старте API.

### Работа с сотрудниками

- создание и редактирование сотрудников;
- назначение ролей `Manager` и `Employee`;
- поиск по списку сотрудников;
- проверка уникальности email;
- защита от удаления собственной учетной записи.

### Работа с проектами

- создание проекта через пошаговый wizard из 5 шагов;
- ввод основной информации о проекте;
- указание компании-заказчика и компании-исполнителя;
- выбор менеджера проекта;
- назначение сотрудников в команду;
- загрузка документов;
- редактирование проекта;
- фильтрация проектов по названию, компании, дате начала и приоритету;
- просмотр прикрепленных документов;
- удаление проекта.

### Работа с задачами

- создание задач;
- привязка задачи к проекту;
- назначение автора и исполнителя;
- изменение статуса задачи (`ToDo`, `In Progress`, `Done`);
- фильтрация по проекту и статусу;
- сортировка;
- проверка уникальности названия задачи в рамках проекта.

## Разграничение доступа

- `Chief`:
  полный доступ к сотрудникам, проектам и задачам.
- `Manager`:
  доступ к своим проектам, возможность редактировать состав команды и назначать исполнителей задач.
- `Employee`:
  просмотр доступных проектов и работа со своими задачами.

## Как запустить проект

### 1. Требования

- .NET SDK 10
- Node.js 24+
- npm 11+
-  Visual Studio

### 2. Запуск backend

Откройте решение через IDE Visual Studio.

Выберите в качестве запускаемого проекта `ProjectManager.API`.

Перейдите в консоль дмчпетчера пакетов, выберите проект по умолчанию `ProjectManager.DAL`.

Создайте миграцию

```powershell
Add-Migration InitialMigration
Update-Database
```

Запустите проект через IDE.

По умолчанию backend стартует на:

- `http://localhost:5224`
- `https://localhost:7267`

### 3. Запуск frontend

Перейдите в папку фронтенда:

```powershell
cd .\projectmanagementfrontend\
npm install
npm start
```

Frontend будет доступен по адресу:

- `http://localhost:3000`
